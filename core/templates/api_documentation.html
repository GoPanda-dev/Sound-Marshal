Sound Marshal API Documentation

Welcome to the Sound Marshal API. This API allows you to interact with the platform programmatically. You can manage user profiles, tracks, transactions, payments, campaigns, submissions, and more.

Authentication

The API uses token-based authentication. To obtain an authentication token, send a POST request with your username and password to the following endpoint:

POST /api-token-auth/

Body Parameters:

  • username: Your username
  • password: Your password

Response:

{ "token": "your_generated_token" }

Include the token in the Authorization header of your requests:

Authorization: Token your_generated_token

API Endpoints

GET /api/profiles/

Retrieve a list of all user profiles.

POST /api/profiles/

Create a new user profile.

GET /api/profiles/{id}/

Retrieve a specific user profile by ID.

PUT /api/profiles/{id}/

Update a specific user profile by ID.

DELETE /api/profiles/{id}/

Delete a specific user profile by ID.

Tracks

GET /api/tracks/

Retrieve a list of all tracks.

POST /api/tracks/

Upload a new track.

Transactions

GET /api/transactions/

Retrieve a list of all transactions.

Payments

GET /api/payments/

Retrieve a list of all payments.

Campaigns

GET /api/campaigns/

Retrieve a list of all campaigns.

POST /api/campaigns/

Create a new campaign.

Submissions

GET /api/submissions/

Retrieve a list of all submissions.

POST /api/submissions/

Create a new submission.
